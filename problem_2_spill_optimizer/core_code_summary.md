# 核心代码总结

## 主要类和方法

### L1UBOptimizedSolver 类

这是解决内存分配问题的主要类，包含了核心算法实现。

#### 核心方法：

1. **solve()** - 主求解方法
   - 读取输入数据（1.json, 2.json）
   - 执行调度优化
   - 计算缓冲区生命周期
   - 执行内存分配算法
   - 生成结果文件

2. **_allocate_buffer()** - 分配缓冲区并更新空闲块列表
3. **_find_best_fit()** - 寻找最佳适配的空闲块
4. **_merge_free_blocks()** - 合并连续的空闲块
5. **_select_spill_candidate()** - 选择最优的spill候选者

### MemoryPreAllocator 类

用于内存预分配分析的辅助类。

#### 核心方法：

1. **calculate_optimal_placement()** - 计算最优的缓冲区放置策略

## 核心算法流程

1. **数据读取** - 从JSON文件读取图数据和调度序列
2. **调度优化** - 通过识别可交换节点对来优化调度序列
3. **生命周期计算** - 计算每个缓冲区的生命周期范围
4. **内存分配** - 使用最佳适配算法进行内存分配
5. **Spill处理** - 当无法分配时，选择合适的缓冲区进行spill操作
6. **结果输出** - 生成多种格式的结果文件

## 关键优化策略

1. **最佳适配算法** - 在空闲块中选择最小且能容纳当前缓冲区的块
2. **动态spill选择** - 基于多种因素（COPY_IN使用情况、大小、生命周期、复用率）选择spill候选者
3. **空闲块合并** - 合并连续的空闲块以提高内存利用率
4. **预分配分析** - 对L1和UB缓存进行预分配分析

## 输出文件

- task2_schedule.json - 优化后的调度序列
- task2_memory.json - 内存分配结果
- task2_spill.json - spill操作记录
- 3.json - 详细的结果报告
